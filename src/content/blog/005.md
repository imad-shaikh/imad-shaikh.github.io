Backpropagation is one of the main algorithms that allows neural networks to learn. At a high level, a neural network makes a prediction, compares that prediction to the correct answer, and then adjusts its internal values so it can make a better prediction next time. Those internal values are the weights and biases of the network.

The purpose of backpropagation is to figure out how each weight and bias contributed to the error. Instead of treating the network like a black box, it works backward from the final prediction and asks a practical question: which parts of the model were responsible for the mistake, and how should they change?

## The Learning Problem

A neural network is made of layers. Information moves forward through the model, from the input layer, through hidden layers, and finally to the output layer. Each connection between neurons has a weight, which controls how strongly one neuron influences another. Each neuron also has a bias, which shifts how easily that neuron activates.

When training begins, these weights and biases are usually not very useful, so the network may produce outputs that are far from the correct answer. To measure how wrong it is, we use a loss function, also called a cost function. The goal of training is to adjust the network's parameters so that this loss gets smaller over time.

That sounds simple, but the scale makes it difficult. A neural network can have thousands, millions, or even billions of parameters. Backpropagation helps determine which parameters mattered most for the error and how they should be adjusted.

## Starting From the Error

Backpropagation begins at the output layer. The model looks at the prediction it made and compares it to the desired output. If an output value was too low, the network needs to push it upward. If an output value was too high, the network needs to push it downward.

The size of the adjustment depends on how wrong the output was. A large error creates a stronger signal, while a small error creates a weaker one. However, the model cannot directly rewrite the final output. That output was created by the weights, biases, and activations in earlier layers, so backpropagation sends the error signal backward to figure out what caused it.

## How Weights and Biases Get Credit

A bias is relatively easy to understand. If a neuron needs to activate more strongly, increasing its bias can help. If it needs to activate less, decreasing its bias can help.

Weights are a little more interesting because their importance depends on the activation coming from the previous layer. If a previous neuron was highly active, then the weight attached to it had more influence on the final result. If the previous neuron barely activated, changing that weight will not matter as much.

This means backpropagation does not update every weight equally. It gives larger updates to weights that had more influence on the error and smaller updates to weights that mattered less. In this sense, backpropagation is a way of assigning responsibility throughout the network.

## Moving Backward Through the Network

Once the output layer's error is understood, the same idea is applied to the previous layer. The network asks how the hidden layer contributed to the output error. Then it asks how the layer before that contributed to the hidden layer's behavior. This continues backward through the entire network.

That is why the algorithm is called backpropagation: the error is propagated backward from the output layer toward the earlier layers. Each layer receives information about how it affected the final loss, and the weights and biases connected to that layer can be adjusted in a direction that should reduce future errors. Gradient descent then uses these calculated adjustments to make small updates to the model.

## Why Mini-Batches Are Used

In theory, a network could calculate updates using the entire training dataset every time. That would give a more accurate estimate of how the model should change, but it is usually too slow.

Instead, training often uses mini-batches, which are smaller groups of training examples. The network calculates an update from one mini-batch, applies that update, and then moves to another mini-batch. Each mini-batch gives an imperfect estimate, but the process is much faster. Over many updates, the model gradually moves toward lower loss.

This is the idea behind stochastic gradient descent and related optimization methods. The steps are not perfectly smooth, but they are efficient enough to make training practical.

## The Big Idea

Backpropagation is not magic. It is a systematic way to measure how much each weight and bias contributed to a mistake.

The model makes a prediction, measures the error, sends that error backward through the network, and updates the parameters that caused it. In simple terms, backpropagation lets a neural network learn from its mistakes by tracing those mistakes backward through the model.
