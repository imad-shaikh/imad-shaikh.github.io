Networking is about how computers communicate with each other. Every time you open a website, send a message, or connect to a server, data has to travel between devices in a structured way.

At the center of this process is the IP address. An IP address is a unique identifier for a device on a network. IPv4 addresses are 32-bit addresses, which allows for about 4 billion possible addresses. Because the number of internet-connected devices has grown so much, the internet has been moving toward IPv6, which uses 128-bit addresses and provides a much larger address space.

When computers communicate, they send data in smaller pieces called packets. Each packet contains an IP header, which includes information such as the sender’s and receiver’s IP addresses. This helps the network deliver the packet to the correct destination. The Internet Protocol defines how this addressing and routing works.

Above the IP layer, applications use their own protocols to format data. For example, web browsers use HTTP or HTTPS so that servers understand what resource is being requested and how to respond.

## TCP and UDP

The transport layer is where TCP and UDP are used.

TCP, or Transmission Control Protocol, provides reliable communication. It makes sure packets arrive completely and in the correct order. TCP uses sequence numbers to track packet order and acknowledgments to confirm delivery. It also uses a process called the three-way handshake to establish a connection before data is exchanged.

This makes TCP useful for situations where accuracy matters, such as loading web pages, downloading files, sending emails, or using SSH.

UDP, or User Datagram Protocol, is faster but less reliable. It does not establish a connection before sending data, and it does not guarantee that packets arrive or stay in order. This makes UDP useful for time-sensitive applications such as video calls, live streaming, and online gaming, where some data loss is usually better than noticeable delay.

The main tradeoff: TCP prioritizes reliability, while UDP prioritizes speed.

## DNS and Domain Names

Most people do not type IP addresses directly into a browser. Instead, they use domain names like github.com or google.com. DNS, or the Domain Name System, translates these human-readable domain names into IP addresses.

When you enter a URL, your browser sends a DNS query to find the IP address for that domain. Once the IP address is found, the browser can connect to the correct server.

DNS uses different types of records. An A record maps a domain name to an IPv4 address, while an AAAA record maps a domain name to an IPv6 address. DNS is coordinated globally by ICANN, and domain registrars such as Namecheap or GoDaddy allow people to register domain names.

## Public IPs, Private IPs, and Ports

Networking also separates public and private IP addresses. Public IP addresses are unique across the internet, while private IP addresses are used inside local networks like homes, schools, or companies.

IP addresses can also be static or dynamic. A static IP address stays the same, which is useful for servers. A dynamic IP address can change over time and is common for home internet connections and everyday devices.

An IP address identifies a device, but a device can run many services at once. Ports identify specific services or processes on that device. For example, port 80 is commonly used for HTTP, port 443 for HTTPS, and port 22 for SSH.

Firewalls help protect networks by monitoring and controlling traffic. They can allow or block traffic based on rules, such as permitting web traffic while blocking ports that should not be publicly accessible.

## Why This Matters

Networking is one of the foundations of modern software. Web applications, APIs, cloud platforms, distributed systems, and remote development tools all rely on these concepts.

Understanding IP addresses, packets, TCP, UDP, DNS, ports, and firewalls gives you a clearer picture of what happens when software communicates across machines. Large systems may be complex, but they are built on top of these same basic networking ideas.
