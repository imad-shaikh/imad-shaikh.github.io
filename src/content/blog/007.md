Before learning about large-scale distributed systems, it helps to understand what is happening inside a single computer. Every application, server, database, or cloud service eventually depends on the same basic hardware ideas: storage, memory, cache, and the CPU.

Computer architecture can get very deep, but the core idea is simple. A computer is built from layers, and each layer has a different job. Some layers are optimized for long-term storage. Others are optimized for speed. Together, they allow programs to store data, access it quickly, and execute instructions.

## Bits, Bytes, and Data

At the lowest level, computers operate using binary: zeros and ones. Each 0 or 1 is called a bit, which is the smallest unit of data in computing.

A byte is made up of eight bits. One byte can represent a small piece of data, such as a character, number, or part of a larger file. From there, data sizes scale upward into kilobytes, megabytes, gigabytes, and terabytes.

This matters because every file, program, image, database record, and network message is ultimately represented as binary data. Higher-level programming languages make this easier to work with, but underneath the abstraction, the machine is still moving and processing bits.

## Disk Storage: Long-Term Memory

Disk storage is where a computer keeps data permanently. This includes the operating system, installed applications, documents, photos, databases, and other files.

Disk storage is non-volatile, which means it keeps data even when the computer is powered off. If you restart your computer, the files on disk are still there.

There are two common types of disk storage: HDDs and SSDs. HDDs use spinning magnetic disks, while SSDs use flash memory. SSDs are usually more expensive, but they are much faster at reading and writing data.

For example, a typical HDD might read data at around 80 to 160 MB/s, while an SSD might range from about 500 MB/s to several thousand MB/s depending on the drive. This speed difference is one reason modern computers and servers often rely on SSDs for better performance.

However, even fast disk storage is still much slower than memory that sits closer to the CPU.

## RAM: Active Working Memory

RAM, or random access memory, is the computer’s active workspace. When a program is running, the data it needs is loaded from disk into RAM so it can be accessed more quickly.

RAM holds things like variables, data structures, application state, intermediate calculations, and the runtime stack. For example, when a program creates an array, stores an object, or calls a function, that information is usually being managed in RAM while the program runs.

Unlike disk storage, RAM is volatile. This means it only keeps data while it has power. When the computer turns off or restarts, the contents of RAM are cleared.

RAM is much faster than disk storage. While a disk may be useful for saving large amounts of data, RAM is better for data that a program needs right now. This is why having more RAM can help a computer handle more applications, larger datasets, or heavier workloads at the same time.

## Cache: Even Faster Memory Near the CPU

Even though RAM is fast, it is still not always fast enough for the CPU. The CPU can execute instructions extremely quickly, so waiting on RAM too often would slow everything down.

This is where cache memory comes in.

Cache is a small amount of very fast memory located close to the CPU. It stores data and instructions that the CPU is likely to use soon. Instead of repeatedly fetching the same data from RAM, the CPU can access it from cache much faster.

Modern CPUs usually have multiple levels of cache:

- L1 cache is the smallest and fastest.
- L2 cache is larger but slightly slower.
- L3 cache is larger still and shared across CPU cores in many processors.

When the CPU needs data, it checks the fastest cache first. If the data is not there, it checks the next cache level, then RAM, and eventually disk if needed. This hierarchy is designed to reduce the average time it takes to access data.

The basic idea is that frequently used data should stay closer to the CPU.

## The CPU: Executing Instructions

The CPU, or central processing unit, is the part of the computer that executes instructions. It performs the actual work of running programs: arithmetic, logic, branching, memory access, and coordination with other parts of the system.

At a high level, the CPU follows a cycle:

- Fetch an instruction.
- Decode what the instruction means.
- Execute the instruction.

When we write code in a high-level language like Python, Java, C, or C++, the CPU cannot directly understand it as written. The code must eventually be translated into instructions the machine can execute.

For compiled languages like C and C++, a compiler converts source code into machine code before execution. For languages like Python, there is usually an interpreter or runtime involved, but the same basic idea remains: the program must eventually become instructions the hardware can run.

The CPU works closely with memory. It reads instructions and data, performs operations, and writes results back. The faster it can access the data it needs, the better the system can perform.

## The Motherboard: Connecting Everything

The motherboard, or main board, connects the major components of the computer. It provides the physical and electrical pathways that allow the CPU, RAM, storage, and other devices to communicate.

Without the motherboard, these components would just be separate pieces of hardware. The motherboard allows data to move between them, making the system function as one machine.

## Why This Matters for System Design

Computer architecture matters because every large system is built on top of these fundamentals.

When a backend service feels slow, the problem might involve disk access, memory usage, CPU bottlenecks, cache misses, or network delays. When a database is under heavy load, performance may depend on how efficiently data moves between disk, memory, and CPU cache. When designing distributed systems, it is easy to focus only on servers, APIs, and databases, but each server is still limited by the hardware underneath it.

Understanding disk storage, RAM, cache, and the CPU gives you better intuition for why systems behave the way they do. It also makes higher-level topics like caching, load balancing, databases, and scalability easier to understand.

A computer is not just "running code" in the abstract. It is constantly moving data through layers of storage and memory, trying to keep the CPU supplied with the information it needs as efficiently as possible. That layered design is one of the foundations of modern computing.
